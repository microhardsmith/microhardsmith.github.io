如何将自己编写的项目上传到github上

2019-07-05

首先，需要安装git，配置好用户名和密码

接下来，你需要在自己的github上new 一个 repository作为你的项目，不要添加readme或其他任何信息

在你的本地项目文件夹中，新建.gitignore文件，在其中写入你不需要在发布中包含的文件

然后根据官方提示的做法：

echo "# 2048AI" >> README.md git init git add README.md git commit -m "first commit" git remote add origin https://github.com/microhardsmith/2048AI.git git push -u origin master

这是发布一个readme文件的流程，我们依此类推

git add . 可以添加目录下所有未被忽略掉的文件

如果你的文件夹中存在超过100m的大文件，那么他会无法提交到远程服务器上，并且在修改提交设置之后也很难进行配置，如果你在之前存在大文件时进行过提交那么你添加gitignore之后也不能成功，需要删除掉以前的commit记录，建议重新来做，删除.git文件夹

git rm -r --cached . 使用这行命令可以清除掉之前的缓存

如果你觉得输命令挺麻烦的，也可以直接将项目clone到电脑上，往文件夹中塞入文件，然后在github desktop中进行提交
